# Requirements: Bookmarks

**Defined:** 2026-05-06
**Core Value:** Users can quickly capture, find, and manage their own bookmarks and related gadgets in one place, with a stable and familiar server-rendered experience — now in their preferred language.

## v1 Requirements

Requirements for milestone v1.11 (Device-aware Font Size Baseline).

### Typography Contract

- [x] **FONT-01**: User can continue choosing exactly three font-size options (Small/Medium/Large) in preferences.
- [x] **FONT-02**: When user selects `medium`, text baseline is device-aware (`PC=14px`, `mobile=16px`).
- [x] **FONT-03**: When user selects `small`, text size is `0.875x` of the device-specific medium baseline.
- [x] **FONT-04**: When user selects `large`, text size is `1.125x` of the device-specific medium baseline.

### Existing-user Migration

- [x] **MIGR-01**: Existing users with `font_size = nil` are migrated to `small`.
- [x] **MIGR-02**: Existing users with `font_size = medium` are migrated to `small`.
- [x] **MIGR-03**: Existing users with `font_size = small` or `large` are not changed by migration, and re-running migration does not change results.

### UX + Safety

- [x] **UX-01**: Users affected by migration receive a one-time in-app notice explaining the baseline change and where to adjust font size.
- [x] **SAFE-01**: Unknown/invalid `font_size` values never crash rendering and always resolve to a safe fallback class.

### Verification

- [x] **TEST-01**: Automated model/helper/controller tests cover canonical mapping and migration behavior contracts.
- [x] **TEST-02**: Automated verification confirms readability contract across modern/classic/simple themes after baseline change.

## v2 Requirements

Deferred to a future milestone.

### Typography Experience

- **UXEX-01**: Show effective current-device size preview text beside font-size selection.
- **UXEX-02**: Add inline helper explanation under font-size selector (device-aware medium and relative small/large behavior).

## Out of Scope

Explicitly excluded from this milestone.

| Feature | Reason |
|---------|--------|
| Separate desktop/mobile font-size settings per user | Increases complexity and UX burden; single symbolic setting is the contract |
| New size levels beyond Small/Medium/Large | Scope expansion beyond agreed 3-option UX |
| Full typography rewrite (`px` -> `rem`) for all theme components | Too broad for this milestone; only contract-critical surfaces are in scope |
| Frontend stack migration (SPA/bundler/device-detection gem) | Conflicts with current Rails/Sprockets constraints |
