# Milestone v1.2: Modern Theme

**Status:** тЬЕ SHIPPED 2026-04-29
**Phases:** 5тАУ9

## Overview

Added a selectable "Modern" theme to the bookmarks app: a hamburger side-drawer navigation and full-page visual polish applied strictly via additive CSS/JS тАФ no new gems, no asset pipeline changes, no HTML structural modifications to existing markup. All new behaviour is gated behind `body.modern` to leave non-modern themes completely unaffected.

**Goal**: The `modern` theme is selectable and active тАФ switching to it applies the `.modern` body class and loads both the CSS and JS scaffolding without breaking any existing theme
**Depends on**: Nothing (first phase of v1.2)
**Requirements**: THEME-01, THEME-02, THEME-03

  3. `menu.js` is compiled and loaded; when `body` does not have class `modern`, the file exits immediately without side effects

## Milestone Summary

**Key Decisions:**

- Strictly additive approach: 2 new files (`themes/modern.css.scss`, `menu.js`) + targeted edits to 3 existing files (layout, menu partial, preferences select). No new gems or npm packages.
- Drawer/backdrop HTML rendered unconditionally in layout; CSS hides them when not in `.modern` theme. Existing `_menu.html.erb` inline `<script>` left intact.
- CSS custom properties use plain hex/rgba literals only (libsass cannot compile `$variable` inside custom property values).
- libsass cannot compile `min(88vw, 320px)` тАФ drawer width uses `width: 88vw; max-width: 320px` with source comment documenting the contract.
- Top-level `.modern`-prefixed selectors (outside `.modern {}` block) used for STYLE-01/03/04 so literal selector strings appear in source for grep-based CI contracts.
- Four new CSS tokens added in Phase 9: `--modern-header-fg`, `--modern-border`, `--modern-surface-alt`, `--modern-danger` тАФ all plain hex.

**Issues Resolved:**

- libsass `min()` incompatibility (Phase 7) тАФ worked around with `max-width`
- Headless Chrome PATH issue in system tests (Phase 8) тАФ resolved in test_helper
- CSS specificity: `.modern #header .head-box` (with ID) required to beat `common.css.scss` `#header .head-box { background: #AAA }` (Phase 9)
- Ruby quoting accident in SCSS contract test (Phase 9, WR-03) тАФ fixed in code review pass

**Technical Debt Incurred:**

- Duplicate `max-width: 100%` in drawer CSS (introduced Phase 7, fixed Phase 9 code review)
- Orphaned `.drawer-overlay { display: block }` block (introduced Phase 7, fixed Phase 9 code review)
- `aria-expanded` on hamburger deferred (NAV-F01, future milestone)
- `box-shadow` middle-bar technique for hamburger has no contract test (Phase 7, IN-01)
- Magic z-index values (1000/1010/1020) have no named tokens (Phase 7, IN-02)
