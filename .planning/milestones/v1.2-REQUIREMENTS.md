## v1.2 Requirements (Final)

### Theme Infrastructure

- [x] **THEME-01**: User can select "modern" theme from the preferences screen
- [x] **THEME-02**: `themes/modern.css.scss` exists with `.modern {}` scope and CSS custom property design tokens
- [x] **THEME-03**: `menu.js` exists with `body.modern` guard for drawer interaction logic

### Navigation

- [x] **NAV-01**: User sees a hamburger button in the header when using the modern theme
- [x] **NAV-02**: User can open a side drawer containing all navigation links by clicking the hamburger
- [x] **NAV-03**: User can close the drawer via backdrop click, Esc key, or clicking a nav link

### Visual Styling

- [x] **STYLE-01**: Modern theme header bar is clean (replaces `#AAA` gray; covers both `#header` and `.header` selectors)
- [x] **STYLE-02**: Modern theme uses 16px base typography with system font stack and improved line-height
- [x] **STYLE-03**: Modern theme tables have modern row styling, padding, and hover states
- [x] **STYLE-04**: Modern theme action buttons and form inputs/selects are visibly styled

### Accessibility

- [x] **A11Y-01**: Drawer slide animation is disabled when `prefers-reduced-motion` is active
