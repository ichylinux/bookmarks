# Milestone v1.3: Quick Note Gadget

**Status:** ✅ SHIPPED 2026-04-30
**Phases:** 10–13
**Total Plans:** 10

## Overview

Add a note-taking tab to the simple theme's welcome page so users can jot and review personal notes without leaving the app. Built entirely on the existing Rails/ActiveRecord/ERB/jQuery/SCSS stack — no new dependencies.

## Phases

### Phase 10: Data Layer

**Goal**: The `notes` table exists with correct schema, the `Note` model enforces ownership and validation, and the foundation for per-user data isolation is in place before any controller or view code is written.
**Depends on**: Phase 9 (existing app baseline)
**Requirements**: NOTE-03
**Plans**: 3 plans

- [x] 10-01-PLAN.md — Migration: create notes table with composite index, run db:migrate
- [x] 10-02-PLAN.md — Note model + notes.yml fixture stub + model unit tests
- [x] 10-03-PLAN.md — User has_many :notes association + routes :notes resource

**Details:**
- `CreateNotes` migration: `user_id` (bigint NOT NULL), `body` (text NOT NULL), `deleted` (boolean default false), `created_at`/`updated_at`, composite index `(user_id, created_at)`
- `Note` model: `Crud::ByUser`, `belongs_to :user`, `before_validation :strip_body`, `validates :body, presence: true, length: { maximum: 4000 }`, `scope :recent`, soft-delete `destroy` override
- `User#has_many :notes, dependent: :destroy` (soft-delete override invoked on user destroy)
- `resources :notes, only: [:create, :destroy]` in `config/routes.rb`

## Milestone Summary

**Key Decisions:**

- Zero new dependencies — entire feature built on existing Rails/ActiveRecord/ERB/jQuery/SCSS stack
- Full-page POST with redirect (not AJAX) for note create — consistent with bookmarks and preferences forms
- Tab switching via jQuery show/hide + `?tab=notes` query param to survive POST/redirect cycle
- Theme isolation: all tab HTML wrapped in `<% if favorite_theme == 'simple' %>`; all tab CSS scoped under `.simple { }` in `welcome.css.scss`
- `user_id` never in strong params — merged from `current_user.id` server-side (pattern from `todos_controller.rb`)
- Rails 8 association `user.notes.delete_all` triggers nullifying updates incompatible with NOT NULL `user_id`; tests use `Note.where(user_id: user.id).delete_all`

**Issues Resolved:**

- Rails 8.1 `has_many ... delete_all` nullify incompatibility with NOT NULL `user_id` — worked around in tests with `Note.where(user_id: user.id).delete_all`
- CSRF: `form_with(local: true)` prevents inline `authenticity_token` copy-paste error from portal layout

**Issues Deferred:**

- Delete individual notes — deferred until core capture flow is confirmed
- Note gadget on modern and classic themes — deferred until simple theme proves out
- `drawer_ui?` extraction as a separate helper concern (todo captured; resolved inline in Phase 13-04)

**Technical Debt Incurred:**

- Phase 10 lacks a formal `10-VERIFICATION.md` (model tests + UAT passed but not promoted to VERIFICATION format)
- Milestone audit was run before phases 12–13 were built; the stale `milestones/v1.3-MILESTONE-AUDIT.md` shows `gaps_found` but was acknowledged at close
