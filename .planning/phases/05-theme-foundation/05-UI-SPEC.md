## Color

These 4 tokens are the complete color contract for Phase 5. They are locked decisions from CONTEXT.md (D-02). All values are plain CSS hex — no SCSS variable references, due to the libsass limitation documented in STATE.md.

| Token | Hex Value | CSS Custom Property | Role |
|-------|-----------|--------------------|----|
| Primary | `#3b82f6` | `--modern-color-primary` | Interactive elements, links, buttons (consumed in Phase 9) |
| Background | `#ffffff` | `--modern-bg` | Page surface / dominant color (consumed in Phase 9) |
| Text | `#1a1a1a` | `--modern-text` | Body copy (consumed in Phase 9) |
| Header background | `#1e40af` | `--modern-header-bg` | Header bar (consumed in Phase 9, STYLE-01) |

**60/30/10 mapping (for Phase 9 reference):**

| Role | Token | Value |
|------|-------|-------|
| Dominant (60%) | `--modern-bg` | `#ffffff` — page background and surfaces |
| Secondary (30%) | `--modern-header-bg` | `#1e40af` — header bar (dark blue) |
| Accent (10%) | `--modern-color-primary` | `#3b82f6` — buttons, links, focus rings |
| Body text | `--modern-text` | `#1a1a1a` — all body copy |

Accent reserved for: primary action buttons, links, focus ring outlines. (Defined now; enforced in Phase 9.)

**Destructive color:** Not defined in this phase. No destructive actions appear in the Phase 5 scope. Add `--modern-color-destructive` in Phase 9 if needed for Delete button styling (STYLE-04).

**SCSS placement contract:**

```scss
// app/assets/stylesheets/themes/modern.css.scss
// Pattern mirrors simple.css.scss: flat .modern {} blocks, no nesting beyond selectors.
.modern {
  --modern-color-primary: #3b82f6;
  --modern-bg:            #ffffff;
  --modern-text:          #1a1a1a;
  --modern-header-bg:     #1e40af;
}
```

All four tokens are declared as direct children of `.modern {}` with no nested selectors in this stub. Values are plain CSS hex strings only — no `$variable` interpolation (libsass bug: SCSS variable references inside CSS custom property values are not compiled correctly).

## JS Guard Contract

`menu.js` is a stub file. The only code required in Phase 5 is the guard pattern. No drawer logic, no event handlers.

```js
// app/assets/javascripts/menu.js
$(function() {
  if (!$('body').hasClass('modern')) return;
  // Drawer interaction logic added in Phase 8.
});
```

Rules:
- Guard is the **first statement** inside the `$(function() {...})` DOM-ready callback.
- When `body` does not have class `modern`, the file exits immediately with zero side effects.
- File must follow v1.1 JS conventions from `.planning/codebase/CONVENTIONS.md` (const/let, `$(function(){})` wrapper).
- Auto-included by `require_tree .` in `application.js` — no manifest change needed.
