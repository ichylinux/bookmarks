Create app/assets/stylesheets/themes/modern.css.scss with exactly this content (per D-01, D-02, UI-SPEC Color section, and PATTERNS.md):

```scss
.modern {
  --modern-color-primary: #3b82f6;
  --modern-bg:            #ffffff;
  --modern-text:          #1a1a1a;
  --modern-header-bg:     #1e40af;
}
```

Critical rules:
- Use plain CSS hex values only. Do NOT use SCSS variable references (e.g., `$primary`) inside custom property values — libsass does not compile them correctly (STATE.md pitfall).
- All 4 tokens are direct children of `.modern {}`. No nested selectors in this stub.
- Token names use the `--modern-` prefix even inside `.modern {}` scope (per D-01 — self-documenting, collision-proof for future themes).
- File has no other content — no comments, no additional rules, no media queries (those belong in Phases 7–9).
- Filename: `modern.css.scss` (not `modern.scss`) — mirrors the `.css.scss` convention of the analog `simple.css.scss`.
- Placed in `app/assets/stylesheets/themes/` — Sprockets auto-includes via `require_tree .` in application.css. No manifest change needed.
