<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Feed records/external entries -> translated UI | Feed titles, URLs, and fetched entries render beside localized fixed labels. |
| Server-rendered attributes -> JavaScript alerts | Translated feed messages cross from ERB attributes into `feeds.js`. |
| Calendar records/holiday_jp -> translated UI | Calendar titles and Japanese holiday names render inside localized calendar chrome. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-04-01 | T | `app/views/feeds/_form.html.erb`, `app/views/welcome/_feed.html.erb` | mitigate | Use Rails-escaped `data-*` attributes or `to_json` for inline JS literals; never concatenate unescaped translation/user content into script. |
| T-17-04-02 | I | Feed/calendar record content | mitigate | Preserve `feed.title`, feed entries, `gadget.title`, `@calendar.title`, and `@calendar.holiday(date)` as content values per D-03/D-04. |
| T-17-04-03 | D | `feeds.js` data key mismatch | mitigate | Use exact names `data-feed-url-required-message` -> `.data('feedUrlRequiredMessage')` and `data-feed-fetch-failed-message` -> `.data('feedFetchFailedMessage')`; cover with integration assertions and ESLint. |
| T-17-04-04 | E | JavaScript i18n pipeline | accept | This plan adds no auth, identity, or new executable dependency surface; it explicitly avoids `i18n-js`, globals, imports, and bundlers. |
</threat_model>
