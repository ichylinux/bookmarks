# Phase 17 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Rails ERB helpers + existing SCSS |
| Icon library | none; existing icon-like glyphs only (`<<`, `>>`, `▶`, `▼`) |
| Font | Inherit existing: common theme user preference sizes; modern theme system stack `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` |

Design system contract:
- Preserve the current Rails server-rendered ERB, SCSS/CSS under `app/assets/stylesheets`, Sprockets, and jQuery architecture.
- Do not add `i18n-js`, shadcn, React, Vite, Next.js, Tailwind, a JavaScript build pipeline, or a new design system.
- Phase 17 may adjust markup attributes or copy placement for translation and accessibility, but it must not restyle the app or introduce new component primitives.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon glyph gaps, compact inline spacing, translated label adjacency |
| sm | 8px | Compact controls, list/card internal spacing, form helper spacing |
| md | 16px | Default element spacing, form groups, gadget content separation |
| lg | 24px | Section padding and dashboard surface separation where existing layout already allows it |
| xl | 32px | Larger layout gaps only when preserving an existing page structure |
| 2xl | 48px | Major section breaks only when already established by surrounding UI |
| 3xl | 64px | Page-level spacing; do not introduce during Phase 17 unless required to prevent translated-copy collision |

Exceptions: none for new Phase 17 changes.

Spacing rules:
- Do not perform a layout or CSS refactor in Phase 17 except to avoid translation overflow or clipped bilingual text.
- Existing legacy CSS values such as 12px, 20px, and 50px may remain untouched, but Phase 17 must not expand their usage.
- English strings may be longer than Japanese strings; allow wrapping in existing cells, gadget bodies, breadcrumbs, and form helper text before adding width changes.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / Meta | 11px | 400 | 1.5 |
| Body | 12px | 400 | 1.5 |
| Heading | 14px | 700 | 1.2 |
| Display / Modern Body | 16px | 400 | 1.5 |

Typography rules:
- Use only 400 and 700 weights for new Phase 17 UI copy.
- Preserve common theme body sizing and user font-size preference behavior: small 10px, medium 12px, large 14px.
- Preserve modern theme 16px body text, line-height 1.5, and existing system font stack.
- Do not add new heading levels, emphasis styles, or decorative text treatments for translated copy.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | App background and primary content surfaces |
| Secondary (30%) | `#f3f4f6` / `#d1d5db` | Alternate surfaces, table headers/stripes, borders, separators |
| Accent (10%) | `#3b82f6` | Modern primary color only: focus outlines, modern links where already established, modern action links, modern gadget title bars |
| Destructive | `#dc2626` | Delete and destructive actions only |

Accent reserved for: modern primary/focus treatments, modern links where already established, modern action links, and modern gadget title bars. Accent is not reserved for every interactive element, and Phase 17 must not recolor legacy/common-theme controls as part of translation work.

Color rules:
- Preserve existing common-theme neutral controls and legacy calendar weekday colors.
- Do not add new decorative color categories for translated empty/error states.
- Error and destructive copy may use existing destructive styling only where delete/destructive behavior already exists.

## Copywriting Contract

Fixed UI copy must be bilingual and must separate app chrome from user/external content.

| Element | Copy |
|---------|------|
| Primary CTA | `ブックマークを追加 / Add Bookmark`; when the surface is not bookmarks, use the surface-specific CTA listed below |
| Empty state heading | Japanese: state what is missing. English: state what is missing. Example: `ブックマークはまだありません / No bookmarks yet` |
| Empty state body | Include the next step. Example: `ブックマークを追加してここに表示します。 / Add a bookmark to show it here.` |
| Error state | Describe problem + solution path. Example: `フィードを取得できませんでした。URLを確認してもう一度お試しください。 / Could not fetch the feed. Check the URL and try again.` |
| Destructive confirmation | Use `messages.confirm_delete` pattern; translate the sentence and interpolate user content unchanged |

Required bilingual action labels:

| Surface | Japanese | English |
|---------|----------|---------|
| Bookmark add | `ブックマークを追加` | `Add Bookmark` |
| Folder create | `フォルダを作成` | `Create Folder` |
| Feed title fetch | `フィードから取得` | `Fetch From Feed` |
| Todo add | `タスクを追加` | `Add Task` |
| Note save | `メモを保存` | `Save Note` |
| Calendar view | `カレンダーを表示` | `View Calendar` |

Copywriting rules:
- Avoid generic CTA labels such as `Submit`, `OK`, and `Cancel`. Use a specific verb + noun.
- Bookmark titles/URLs, folder names, note bodies, todo titles, feed site names/entries, calendar record titles, and Japanese holiday names are user/external content and must remain unchanged.
- Fixed gadget names localize: `BookmarkGadget#title` and `TodoGadget#title`.
- Feed and calendar gadget titles are user-created records and must remain unchanged.
- Todo priority labels localize at display time; stored numeric values remain unchanged.
- Calendar month/year and weekday labels localize; `holiday_jp` holiday names remain Japanese and intentionally untranslated.
- User/external content may be interpolated into translated sentences, but translation keys must never be built from user/external content.

## Visual Hierarchy

Primary visual anchor remains the existing gadget title/header/action areas:
- Dashboard gadgets keep their existing title bars and content bodies. Modern theme gadget title bars may continue using `#3b82f6`; common theme title bars keep existing neutral styling.
- Bookmark breadcrumbs and action areas remain the navigation/action anchor for bookmark screens.
- Feed/calendar/todo/note surfaces keep existing form controls, table/list structures, and gadget layouts.

Hierarchy contract:
- User content remains visually primary inside lists/cards: bookmark titles, note bodies, todo titles, feed entries, calendar titles, and holiday names must not be visually demoted because surrounding labels are translated.
- Fixed UI labels should clarify controls without competing with content.
- Translation must not introduce new decoration, iconography, badges, color chips, or card treatments.
- If translated text is longer, prefer wrapping, concise copy, or existing helper text placement over adding new visual containers.

---

## Interaction and Accessibility

Interaction contract:
- JavaScript-visible messages must be supplied through server-rendered translated values, preferably `data-*` attributes on the relevant button/container, or escaped ERB values when an inline script is already present.
- Do not add a global JavaScript translation registry, `window.I18n`, `i18n-js`, or a JavaScript build step.
- Existing jQuery/Sprockets interactions stay in place; Phase 17 only replaces hardcoded prose with server-provided translated values.
- Icon-like controls `<<`, `>>`, `▶`, and `▼` remain visual glyphs. They do not need translation unless Phase 17 adds accessible labels.

Accessibility contract:
- If glyph-only controls receive labels, labels must be localized via Rails I18n, for example previous/next month labels for `<<` and `>>`.
- Preserve the rendered `<html lang>` behavior shipped in Phase 14.
- Maintain existing modern theme `:focus-visible` outlines using `#3b82f6`.
- Do not remove visible text labels in favor of icons.
- Empty states must provide a next step; error states must provide a problem and a recovery path.
