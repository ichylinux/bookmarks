# Plan 01 — `var` 排除と `const` / `let` への置換（STYL-01）

<objective>
第一者 `app/assets/javascripts/**/*.js` から `var` を除去し、`@eslint/js` recommended の `no-var` を満たす。Sprockets のグローバル名前空間（`todos` / `feeds` / `calendars`）は `window.*` 連携手順で**同一オブジェクト**を指す。jQuery の `this` 束縛に触れる箇所は**変換しない**（本プランは `var`→`const`/`let` のみ; STYL-02 の細部は Plan 02）。
</objective>

<threat_model>
| ID | 脅威 | 深刻度 | 緩和（本プラン） |
|----|------|--------|------------------|
| T-3-01 | 誤置換でグローバル束縛が壊れ UJS/ガジェットが沈黙失敗 | medium | `window.ns = window.ns \|\| {}` + `const ns = window.ns` パターン; 変更後に `yarn run lint` 必須 |
| T-3-02 | 誤ったアロー化で jQuery `this` が壊る | high | 本プランでは **コールバック形を変えない**（`var` 以外を触るのは定数化のみ） |
| T-3-03 | クライアント内の挙示差分で本番 XSSI 等 | low | 本範囲はスタイルのみ; 挙動変更は行わない |
</threat_model>

<must_haves>
- コマンド `yarn run lint` が exit code 0
- `app/assets/javascripts` 下の `*.js` について、`\bvar\b` が **ゼロ**（`eslint.config.mjs` の `ignores` 対象外）。例外は Plan で明示する `// eslint-disable-next-line no-var` + 1 行理由のみ
- `cable.js` / `application.js` を不要に編集しない
</must_haves>
