## DOM Structure Contract

This section is Phase 6-specific. It documents the exact HTML surface that downstream phases
(7: CSS, 8: JS) depend on. This is the primary design contract for this phase.

### Hamburger Button

```
Selector:  button.hamburger-btn
Placement: Last child of div.head-box (inside #header)
Content:   Empty — no child elements, no text
ARIA:      aria-label="メニュー"
Guard:     Unconditional — rendered for all users, all themes
Phase 7:   display: none by default; display: block under .modern
Phase 7:   ::before, ::after, box-shadow draw the three horizontal lines
Phase 7:   Transforms to X icon when body.drawer-open is present
```

### Drawer Panel

```
Selector:  div.drawer
Placement: Direct child of <body>, after closing </div> of .wrapper, before </body>
Content:   <nav> containing 7 link_to calls (see Nav Links below)
Guard:     Inside if user_signed_in? — not rendered for guests
Phase 7:   Positioned off-screen left by default; slides in on .drawer-open
Phase 8:   body.drawer-open class toggled by hamburger click
```

### Overlay

```
Selector:  div.drawer-overlay
Placement: Direct child of <body>, sibling of div.drawer, before </body>
Content:   Empty
Guard:     Inside same if user_signed_in? block as drawer
Phase 7:   Transparent/hidden by default; semi-opaque backdrop on .drawer-open
Phase 8:   Click handler closes drawer (removes body.drawer-open)
```

### Nav Links (drawer content — all 7 required)

Order matches `_menu.html.erb` exactly:

| Order | Label | Path Helper | Method |
|-------|-------|-------------|--------|
| 1 | Home | `root_path` | GET |
| 2 | 設定 | `preferences_path` | GET |
| 3 | ブックマーク | `bookmarks_path` | GET |
| 4 | タスク | `todos_path` | GET |
| 5 | カレンダー | `calendars_path` | GET |
| 6 | フィード | `feeds_path` | GET |
| 7 | ログアウト | `destroy_user_session_path` | DELETE (`method: 'delete'`) |

Inner wrapper: `<nav>` element (semantically correct for navigation landmark; gives Phase 7 the `.drawer nav` selector).

Source: CONTEXT.md D-06, `app/views/common/_menu.html.erb` — verified 2026-04-29.

## Interaction States

Phase 6 defines HTML structure only. Interaction states are Phase 7 (CSS) and Phase 8 (JS)
concerns. Documented here for completeness so downstream phases have the full state model.

| State | Trigger | DOM Change | Visual Change |
|-------|---------|------------|---------------|
| Drawer closed (default) | Page load | No `drawer-open` on `<body>` | Drawer off-screen, overlay hidden (Phase 7) |
| Drawer open | Hamburger click (Phase 8) | `body.drawer-open` added | Drawer slides in, overlay appears (Phase 7) |
| Drawer closing | Backdrop click / Esc / nav link (Phase 8) | `body.drawer-open` removed | Drawer slides out, overlay fades (Phase 7) |
| Hamburger default | No `drawer-open` | — | Three-line icon (Phase 7 CSS) |
| Hamburger active | `body.drawer-open` present | — | X icon (Phase 7 CSS transform) |
| Reduced motion | `prefers-reduced-motion: reduce` | — | Instant show/hide, no transition (Phase 7) |
