## Rationale

- `session[:otp_user_id]` is reused only as server-side pending 2FA state that was set after password validation.
- The implementation does not call `sign_in` before OTP validation succeeds.
- Pending user locale remains only a candidate; `Preference::SUPPORTED_LOCALES.include?(candidate.to_s)` still gates every locale before `I18n.with_locale`.
- Stale unsupported saved locales fall through to existing fallback behavior.
- Regression tests cover both saved English pending OTP rendering and unsupported saved-locale fallback.

## Residual Risk

Low. The additional lookup is limited to requests carrying `session[:otp_user_id]` and affects locale selection only, not authorization or data access.
