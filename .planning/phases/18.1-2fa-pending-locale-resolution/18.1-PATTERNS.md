# Phase 18.1 Patterns — 2FA Pending Locale Resolution

## Scope

This phase closes a single milestone-audit gap. It should not introduce new UI, routing, database schema, or authentication state transitions.

## Target Files

| File | Role | Pattern to Preserve |
|------|------|---------------------|
| `app/controllers/concerns/localization.rb` | Locale resolution source of truth | Keep `around_action :set_locale` and whitelist every locale candidate before `I18n.with_locale` |
| `app/controllers/users/sessions_controller.rb` | Creates pending 2FA session | Continue storing `session[:otp_user_id]` after password validation, without signing in early |
| `app/controllers/users/two_factor_authentication_controller.rb` | OTP challenge and verification | Keep `skip_before_action :authenticate_user!`; pending user is identified only by `session[:otp_user_id]` |
| `test/controllers/two_factor_authentication_controller_test.rb` | Regression coverage | Use existing integration-test style: enable 2FA, POST sign-in, GET OTP page, assert `html[lang]` and translated label |
| `test/controllers/application_controller_test.rb` | Locale fallback guardrails | Existing tests protect signed-in saved locale, Accept-Language, unsupported fallback, and default Japanese behavior |

## Existing Locale Resolution Pattern

```ruby
def resolved_locale
  [saved_locale, accept_language_match].each do |candidate|
    return candidate.to_sym if candidate && Preference::SUPPORTED_LOCALES.include?(candidate.to_s)
  end
  I18n.default_locale
end
```

The whitelist in `resolved_locale` is the final safety gate. New pending-OTP lookup code should return a stored locale candidate, not call `I18n.with_locale` directly.

## Current Gap

```ruby
def saved_locale
  return nil unless user_signed_in?
  current_user.preference&.locale
end
```

During OTP challenge, `user_signed_in?` is false, but `session[:otp_user_id]` contains the user id. The saved account locale is therefore skipped.

## Implementation Pattern

Use a small helper rather than changing sign-in timing:

```ruby
def saved_locale
  saved_locale_user&.preference&.locale
end

def saved_locale_user
  return current_user if user_signed_in?
  return nil if session[:otp_user_id].blank?

  User.find_by(id: session[:otp_user_id])
end
```

Key constraints:

- Do not call `sign_in` before OTP validation.
- Do not trust params or headers as saved-locale inputs.
- Do not bypass `Preference::SUPPORTED_LOCALES.include?`.
- If `session[:otp_user_id]` is stale or invalid, return nil so `Accept-Language` and `:ja` fallback keep working.

## Regression Test Pattern

Add one focused test to `test/controllers/two_factor_authentication_controller_test.rb`:

```ruby
def test_保存済みlocaleがenのユーザはOTPページを英語で表示する
  user.preference.update!(locale: 'en')
  user.enable_two_factor!

  post user_session_path, params: { user: { email: user.email, password: 'testtest' } }
  assert_redirected_to users_two_factor_authentication_path

  get users_two_factor_authentication_path
  assert_response :success
  assert_select 'html[lang=?]', 'en'
  assert_select 'label', text: I18n.t('two_factor.code_label', locale: :en)
end
```

This intentionally omits `Accept-Language` on both requests, proving saved preference drives the OTP page.
