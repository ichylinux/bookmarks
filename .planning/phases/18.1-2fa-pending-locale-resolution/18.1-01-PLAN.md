<objective>
Close the v1.4 milestone audit gap where a user's saved locale does not reach the 2FA OTP
challenge page after password authentication and before OTP completion.

The fix is deliberately small: teach the existing `Localization` concern to consider the
pending OTP user identified by `session[:otp_user_id]` when no Devise user is signed in.
Keep all Phase 14 invariants: `around_action`, `I18n.with_locale`, and the
`Preference::SUPPORTED_LOCALES` whitelist remain the only path to setting `I18n.locale`.

Output:
- `app/controllers/concerns/localization.rb` updated with pending-OTP saved-locale lookup
- `test/controllers/two_factor_authentication_controller_test.rb` extended with saved-locale OTP regression coverage
</objective>

Current `Localization#resolved_locale`:

```ruby
def resolved_locale
  [saved_locale, accept_language_match].each do |candidate|
    return candidate.to_sym if candidate && Preference::SUPPORTED_LOCALES.include?(candidate.to_s)
  end
  I18n.default_locale
end
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser session -> Rails controller | `session[:otp_user_id]` is server-side session state set only after password validation |
| Localization concern -> User lookup | Pending locale lookup reads a user id from session and queries `User.find_by` |
| Stored preference -> I18n locale | Stored locale is still validated by `Preference::SUPPORTED_LOCALES` before `I18n.with_locale` |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-18.1-01 | S — Spoofing | `session[:otp_user_id]` lookup | Low | mitigate | Do not add any params/header path for setting pending user id; only reuse existing server-side session created after password validation |
| T-18.1-02 | E — Elevation of privilege | Pending OTP flow | Medium | mitigate | Do not call `sign_in` from `Localization`; `Users::TwoFactorAuthenticationController#verify` remains the only OTP completion path |
| T-18.1-03 | T — Tampering | Stored locale candidate | Low | mitigate | Keep `Preference::SUPPORTED_LOCALES.include?(candidate.to_s)` in `resolved_locale`; stale unsupported values fall through to safe fallback |
| T-18.1-04 | I — Information disclosure | Pending user lookup timing | Low | accept | Lookup uses an already authenticated pending session id and only affects UI language, not data access |
</threat_model>
