Implement the server-rendered simple-theme note gadget inside the existing `#notes-tab-panel`: `WelcomeController#index` prepares only current-user note data, `welcome/index` renders a dedicated `_note_gadget.html.erb` partial, and `welcome.css.scss` adds note styles only under the existing `.simple { }` scope.

- Empty state text is exactly `メモはまだありません`.

</must_haves>

Render the body with `<%= note.body %>` so Rails escapes user input. Do not use `raw`, `html_safe`, markdown rendering, autosave JavaScript, AJAX, Turbo, pagination, delete links, or edit controls.

<threat_model>
- XSS: Note body is user input. Render with `<%= note.body %>` and CSS `white-space: pre-wrap`; never use `raw`, `html_safe`, or markdown rendering.
- User isolation: `WelcomeController#index` must query `current_user.notes.recent`; global `Note` queries can leak another user's notes.
- CSRF: Use Rails `form_with ... local: true` so authenticity token handling remains Rails-managed.
- Theme exposure: The partial is mounted only inside the existing `favorite_theme == 'simple'` branch; modern/classic pages must not render `#notes-tab-panel`.
</threat_model>
