Create `features/step_definitions/notes.rb` with four Japanese step definitions matching the feature text.

Implementation requirements:

1. Step `シンプルテーマでサインインします。`
   - Run `user.preference.update!(theme: 'simple')`.
   - Clean current user's notes with `user.notes.delete_all` to keep first-item assertions deterministic.
   - Call `sign_in user`.

2. Step `ルートページでノートタブを開きます。`
   - `visit root_path`
   - Activate the tab with `find('button.simple-tab[data-simple-tab="notes"]').click`
   - Assert the note panel is visible with `assert has_selector?('#notes-tab-panel')`.
   - Assert active tab state with `assert has_selector?('button.simple-tab.simple-tab--active[data-simple-tab="notes"]')`.
   - Use `capture` after the assertions if matching existing visual trace style.

3. Step `/^メモに (.*?) と入力して保存します。$/`
   - Store the body in an instance variable, e.g. `@note_body = body`.
   - Within `#notes-tab-panel`, fill the textarea using `find('textarea[name="note[body]"]').set(body)` or `fill_in 'メモ', with: body`.
   - Click `保存`.
   - Assert `assert_equal root_path, current_path`.
   - Assert the query includes `tab=notes`. Use `Rack::Utils.parse_query(URI.parse(current_url).query)['tab']` or an equivalent parser and assert it equals `'notes'`.

4. Step `/^ノートタブに (.*?) が先頭表示されます。$/`
   - Assert the expected body matches `@note_body` when present.
   - Assert `assert has_selector?('button.simple-tab.simple-tab--active[data-simple-tab="notes"]')`.
   - Scope to `within '#notes-tab-panel'`.
   - Assert `assert has_selector?('.note-item:first-child .note-body', text: body)`.

Prefer stable selectors over visible tab text because the current tab labels are `Home` and `Note`. Do not use sleep. Do not add JavaScript beyond the existing tab behavior.
  </action>
