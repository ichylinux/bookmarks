**Step 2 — 空文字 → nil 正規化を `#update` および `#create` の両方に追加。**

D-05 の通り、`<option value="">自動</option>` から HTTP には `locale=""` が submit される。`presence` で空文字 → nil 正規化してから ActiveRecord に渡す必要がある。

`#update` メソッドを以下のように書き換える:

```ruby
def update
  @user = User.find(current_user.id)
  attrs = user_params
  if attrs[:preference_attributes].respond_to?(:key?) && attrs[:preference_attributes].key?(:locale)
    attrs[:preference_attributes][:locale] = attrs[:preference_attributes][:locale].presence
  end
  @user.attributes = attrs

  @user.transaction do
    @user.save!
  end

  redirect_to preferences_path
end
```

`#create` メソッドも同形に揃える（`redirect_to action: 'index'` は Rails 8 で `preferences_path` と等価だが、明示的に揃えて読みやすくする）:

```ruby
def create
  @user = User.find(current_user.id)
  attrs = user_params
  if attrs[:preference_attributes].respond_to?(:key?) && attrs[:preference_attributes].key?(:locale)
    attrs[:preference_attributes][:locale] = attrs[:preference_attributes][:locale].presence
  end
  @user.attributes = attrs

  @user.transaction do
    @user.save!
  end

  redirect_to preferences_path
end
```

**重要な実装ノート**:
- `attrs[:preference_attributes]` は `ActionController::Parameters` インスタンス。`String#presence` は空文字 `""` → `nil`、`"ja"` → `"ja"` を返す。
- `respond_to?(:key?)` ガードは preference_attributes 自体が無いケース (e.g. 名前のみ更新) でも安全に動作させるため。
- `.key?(:locale)` ガードは locale フィールドが submit されないケース（古い form session など）で `nil[:locale]` を回避するため。
- 既存挙動への影響: `theme`, `font_size` 等の他フィールドは `presence` を経由しない → 既存テスト全 green を維持。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP form → controller | `<option value="">自動</option>` から送信される空文字を controller が信頼してはいけない（DB 検証 inclusion で reject される） |
| controller → model | strong params で `:locale` を許可、`presence` で空文字 → nil 正規化、ActiveRecord validation が SUPPORTED_LOCALES inclusion で最終ガード |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | `user[preference_attributes][locale]` form field | mitigate | strong params で `:locale` のみを permit。配列・hash・nested object の流入は `permit` が拒否。Rails 標準 |
| T-15-02 | Tampering | DB に空文字 `""` が書き込まれる | mitigate | controller `presence` 正規化で空文字 → nil。仮に通過しても model `validates :locale, inclusion: in: SUPPORTED_LOCALES` で `''` は in に含まれず reject (Phase 14 既存検証) |
| T-15-03 | Tampering | 不正 locale 値 `'fr'`, `'<script>'` 等の流入 | mitigate | model validation で SUPPORTED_LOCALES (`%w[ja en]`) inclusion 強制。Phase 14 で confirmed test green |
| T-15-04 | Information Disclosure | redirect_to で別ユーザの preferences が見える | accept | 個人アプリ + `current_user` セッション固定 + `User.find(current_user.id)`。Devise が認証境界を保証、横断アクセスは構造的に不可能 |
| T-15-05 | Denial of Service | locale フィールドの巨大文字列 (length 10MB 等) を post | mitigate | model validation で inclusion チェック前に既に reject (Rails の許容 string パラメータサイズが Rack 経由で先に制限される)。追加ガード不要 |
</threat_model>
