```ruby
require 'test_helper'

class ApplicationControllerTest < ActionDispatch::IntegrationTest

  # VERI18N-01: 保存済みlocaleの経路
  def test_保存済みlocaleがenのユーザは英語でレンダリングされる
    user.preference.update!(locale: 'en')
    sign_in user
    get root_path
    assert_response :success
    assert_select 'html[lang=?]', 'en'
  end

  # VERI18N-01: Accept-Language経路
  def test_AcceptLanguageがenのリクエストは英語でレンダリングされる
    user.preference.update!(locale: nil)
    sign_in user
    get root_path, headers: { 'Accept-Language' => 'en-US,en;q=0.9,ja;q=0.8' }
    assert_response :success
    assert_select 'html[lang=?]', 'en'
  end

  # VERI18N-01: 無効locale (I18N-03)
  def test_無効なlocaleは無視されてデフォルトの日本語になる
    user.preference.update!(locale: nil)
    sign_in user
    get root_path, headers: { 'Accept-Language' => 'fr-FR,fr;q=0.9' }
    assert_response :success
    assert_select 'html[lang=?]', 'ja'
  end

  # VERI18N-01: デフォルト :ja
  def test_locale未設定かつAcceptLanguage未指定の場合はデフォルト日本語
    user.preference.update!(locale: nil)
    sign_in user
    get root_path
    assert_response :success
    assert_select 'html[lang=?]', 'ja'
  end

end
```

**Test design notes:**

- Each test calls `user.preference.update!(locale: ...)` at the start to ensure clean, independent fixture state. Tests cannot share mutable locale state.
- `user` returns fixture user 1 (memoized `User.first` from test/support/users.rb).
- `user.preference` for fixture user 1 returns the persisted preference record (user_id: 1) from preferences.yml.
- `sign_in user` authenticates via Devise::Test::IntegrationHelpers (included in test_helper.rb line 18 for ActionDispatch::IntegrationTest).
- `get root_path` hits WelcomeController#index — a signed-in-required action that renders application.html.erb, which now contains `<html lang="<%= I18n.locale %>">`.
- `assert_select 'html[lang=?]', 'en'` verifies the rendered `<html lang>` attribute directly (I18N-04 evidence).
- Test 1 (saved locale) sets locale to 'en' — exercises Localization#saved_locale path.
- Test 2 (Accept-Language) sets locale to nil, provides `en-US,en;q=0.9` header — exercises `accept_language_match` path.
- Test 3 (invalid fallback) sets locale to nil, provides `fr-FR` header — exercises full pipeline fall-through to default :ja.
- Test 4 (default) sets locale to nil, no Accept-Language — exercises the default :ja return from resolved_locale.
