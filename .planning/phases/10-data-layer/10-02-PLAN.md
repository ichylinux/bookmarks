Create `app/models/note.rb` with EXACTLY this content:

```ruby
class Note &lt; ActiveRecord::Base
  include Crud::ByUser

  belongs_to :user

  before_validation :strip_body

  validates :body, presence: true, length: { maximum: 4000 }

  scope :recent, -&gt; { order(created_at: :desc) }

  def destroy
    update!(deleted: true)
  end

  private

  def strip_body
    body&amp;.strip!
  end
end
```

Specific requirements (verbatim from CONTEXT.md decisions and PATTERNS.md final shape):
- Inherit from `ActiveRecord::Base` (matches `Todo`/`Bookmark`/`Calendar`), NOT `ApplicationRecord`. The closest analog is `Todo`.
- `include Crud::ByUser` — gives `readable_by?`/`updatable_by?`/`deletable_by?` for NOTE-03 ownership enforcement. Path is `app/models/crud/by_user.rb` (NOT `app/models/concerns/crud/by_user.rb` — RESEARCH.md correction; Rails autoloading resolves `Crud::ByUser` from `app/models/crud/`).
- `belongs_to :user` — Rails 5+ adds presence validation automatically; do NOT add `validates :user, presence: true`.
- `before_validation :strip_body` + private `strip_body` method calling `body&.strip!` — strips in place so whitespace-only fails the presence check. Use the named-method form (greppable), not the inline block form.
- `validates :body, presence: true, length: { maximum: 4000 }` — both validations on one line per Rails idiom. The 4000 cap is well below MySQL TEXT's 65535 byte limit; protects against DoS per CONTEXT.md.
- `scope :recent, -> { order(created_at: :desc) }` — explicit named scope. Do NOT use `default_scope` — CONTEXT.md and RESEARCH.md Pitfall 2 explicitly forbid it (leaks into counts/joins/fixtures).
- `def destroy; update!(deleted: true); end` — soft-delete override. Mirrors `Bookmark#destroy_logically!` body but renamed to `destroy` so `User.has_many :notes, dependent: :destroy` triggers soft-delete. Do NOT define `destroy_logically!` (already provided by `daddy` gem on every AR::Base).
- Do NOT define `scope :not_deleted` — already provided gem-wide by `Daddy::Models::QueryExtension`.
- Do NOT add a `creatable_by?` method — `Crud::ByUser` does not define it, and the test must not assert it.

```ruby
require 'test_helper'

class NoteTest &lt; ActiveSupport::TestCase

  def test_他人のメモは参照できない
    user = User.find(2)
    note = Note.where('user_id &lt;&gt; ?', user.id).first
    assert note, 'fixture should provide a note owned by a different user'
    assert ! note.readable_by?(user)
    assert ! note.updatable_by?(user)
    assert ! note.deletable_by?(user)
  end

  def test_自分のメモは参照できる
    user = User.find(1)
    note = Note.where(user_id: user.id).first
    assert note
    assert note.readable_by?(user)
    assert note.updatable_by?(user)
    assert note.deletable_by?(user)
  end

  def test_body_is_required
    note = Note.new(user_id: 1, body: '')
    assert_not note.valid?
    assert note.errors[:body].any?
  end

  def test_whitespace_only_body_fails_presence
    note = Note.new(user_id: 1, body: '   ')
    assert_not note.valid?
    assert note.errors[:body].any?
  end

  def test_body_max_length_4000
    note = Note.new(user_id: 1, body: 'a' * 4001)
    assert_not note.valid?
    assert note.errors[:body].any?
  end

  def test_body_at_4000_chars_is_valid
    note = Note.new(user_id: 1, body: 'a' * 4000)
    assert note.valid?, note.errors.full_messages.inspect
  end

  def test_recent_scope_orders_by_created_at_desc
    sql = Note.recent.to_sql
    assert_match(/ORDER BY.*created_at.*DESC/i, sql)
  end

  def test_destroy_soft_deletes
    note = notes(:one)
    note.destroy
    reloaded = Note.find(note.id)
    assert reloaded.deleted, 'destroy should set deleted: true (soft-delete), not remove the row'
  end

end
```
