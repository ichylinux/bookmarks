# Phase 18.2 Plan 01: Preferences Locale Change Flash Summary

Translate the `preferences.saved` flash under the just-saved candidate locale via
`I18n.with_locale`, gated by `Preference::SUPPORTED_LOCALES`, so the redirected
`/preferences` page no longer renders a mixed-locale flash after a language change.

## Gap Closed

`preferences-locale-change-flash-old-locale` from `.planning/v1.4-MILESTONE-AUDIT.md`.

Before this plan, `PreferencesController#create` and `#update` materialized
`flash[:notice] = t('preferences.saved')` while the request was still running under
the OLD `I18n.locale` (resolved by `Localization#set_locale` before save). The
redirected GET re-resolved the locale correctly for `<html lang>`, but the flash
string was already serialized in the old language, producing a mixed-locale page.

## Implementation Notes

`app/controllers/preferences_controller.rb:36-44` (helper):

```ruby
def saved_notice
  candidate = @user.preference&.locale
  if candidate.present? && Preference::SUPPORTED_LOCALES.include?(candidate.to_s)
    I18n.with_locale(candidate.to_sym) { t('preferences.saved') }
  else
    t('preferences.saved')
  end
end
```

Constraints honored:

- `Localization` concern (`app/controllers/concerns/localization.rb`) is unchanged.
  The `around_action :set_locale` and `Preference::SUPPORTED_LOCALES` whitelist
  remain the request-locale resolver; this helper only scopes the flash string
  translation, it does not mutate `I18n.locale` for the request.
- Helper reads `@user.preference&.locale` (the just-saved value), not raw params,
  so non-locale-touching saves (font_size only, default_priority only, etc.)
  translate the flash under the existing persisted locale and behave identically
  to the prior implementation.
- Helper falls back to plain `t('preferences.saved')` whenever the candidate is
  blank or fails the `SUPPORTED_LOCALES.include?(candidate.to_s)` whitelist gate
  — preserving Phase 14 invariants and avoiding any path where untrusted input
  reaches `I18n.with_locale`.

Threat T-18.2-01 (Tampering / I18n key injection via locale candidate) is mitigated
by the explicit whitelist gate before `I18n.with_locale`. Threat T-18.2-02 is
mitigated by the existing `validates :locale, inclusion: { in: SUPPORTED_LOCALES }`
on the model — the helper reads back the validated persisted value.
